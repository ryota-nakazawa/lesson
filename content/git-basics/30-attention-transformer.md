# AttentionとTransformer

現在のLLMを理解するうえで、AttentionとTransformerは重要なキーワードです。

細かい数式まで覚える必要はありません。まずは「文章の中で、どの言葉に注目するかを扱う仕組み」と「それを中心に作られたモデルの構造」として押さえます。

> まとめ: Attentionは重要な文脈に注目する仕組み、TransformerはAttentionを中心にした現代的なAIモデルの構造です。

## 生成AIまでの大まかな流れ

生成AIは突然出てきたものではなく、いくつかの技術の積み重ねで発展してきました。

| 技術 | ざっくりした役割 |
| --- | --- |
| ルールベース | 人間が条件やルールを書く |
| 機械学習 | データからパターンを学ぶ |
| 深層学習 | 多層のニューラルネットワークで複雑な特徴を学ぶ |
| RNN | 時系列や文章のような順序データを扱うために使われた |
| Attention | 入力のどこに注目するかを学ぶ |
| Transformer | Attentionを中心にした構造で、大規模な言語モデルの土台になった |

RNNは、文章を前から順番に処理する技術として使われてきました。その後、AttentionとTransformerによって、大量の文章を扱うLLMが大きく発展しました。

CNNは画像認識などで重要な技術ですが、このレッスンの主題は「文章の文脈をどう扱うか」です。そのため、ここではRNN、Attention、Transformerに絞って理解します。

## Attentionとは

Attentionは、入力の中で重要な部分に重みを置く仕組みです。

たとえば、次の文章を考えます。

```txt
田中さんは佐藤さんに本を渡しました。彼はとても喜びました。
```

このとき「彼」が誰を指すのかを考えるには、前の文脈を見る必要があります。

Attentionは、文章中の単語同士の関係を見ながら、回答や次の単語を作るときにどこへ注目するかを扱います。

```txt
入力文
  ↓
どの単語同士が関係しているかを見る
  ↓
文脈に合う出力を作る
```

## 翻訳でなぜ効いたのか

Attentionは、機械翻訳の精度向上にも大きく関係しました。

以前の翻訳モデルでは、RNNを使って文を前から順番に処理し、入力文を圧縮した情報から翻訳文を作る方式がよく使われていました。

この方式では、文が長くなったり、日本語と英語のように語順が大きく違ったりすると、重要な情報をうまく保つのが難しくなります。

Attentionを使うと、翻訳文を作るデコーダが、入力文のどの単語や表現に注目すべきかを毎回見に行けるようになります。

```txt
入力文: 私は 昨日 友だち と 映画 を 見た
  ↓
翻訳するときに、必要な単語へ注目する
  ↓
I watched a movie with my friend yesterday
```

語順が違う言語でも、出力する単語ごとに入力文の関連する部分を参照しやすくなるため、文脈を踏まえた翻訳がしやすくなりました。

Google翻訳でも、ニューラル機械翻訳とAttentionの考え方が取り入れられたことで、従来より自然な翻訳ができるようになったと言われています。

> ここで大事なのは、Attentionによって「前から順番に読んだ情報だけに頼る」のではなく、「必要な文脈を見に行く」考え方が強くなったことです。

## Transformerとは

Transformerは、Attentionを中心にしたモデルの構造です。

RNNのように文章を前から順番に処理するだけでなく、文章全体の関係を見ながら処理しやすい点が特徴です。

| 観点 | RNNのイメージ | Transformerのイメージ |
| --- | --- | --- |
| 処理 | 順番に読む | 文全体の関係を見やすい |
| 長い文脈 | 苦手になりやすい | 比較的扱いやすい |
| 大規模化 | 難しい面がある | 大量データで学習しやすい |

現在の多くのLLMは、Transformer系の構造を土台にしています。

## なぜLLMに効いたのか

Transformerは、大量の文章データから言葉の関係を学ぶのに向いていました。

そのため、次のようなことが可能になりました。

- 文脈に合う文章を生成する
- 長い文章の関係を扱う
- 翻訳、要約、質問応答などを同じモデルで扱う
- 大量のデータと計算資源で性能を伸ばす

LLMでは、文章をトークンに分け、次に来そうなトークンを予測します。

このときTransformerは、直前の単語だけでなく、文中のどの単語や文脈が次の予測に関係するかを見ます。

```txt
入力: 昨日、雨が降っていたので、私は傘を
  ↓
文脈を見る
  ↓
次に来そうなトークン: さした / 持った / 買った ...
```

つまり、Transformerは「文脈を見たうえで、次に来る単語を統計的に判断する」ための土台になっています。

もちろん、Transformerだから必ず正しい回答になるわけではありません。仕組みとして文脈を扱いやすくなった一方で、事実確認は必要です。

## まず押さえること

- Attentionは、入力の中で重要な部分に注目する仕組み
- Attentionにより、翻訳時に入力文の必要な部分を参照しやすくなった
- Transformerは、Attentionを中心にしたモデル構造
- RNNは順番に処理するため長い文脈や語順の違いが難しくなりやすい
- Transformerは、文脈を見ながら次に来るトークンを予測するLLMの土台になった
- TransformerはLLM発展の大きな土台になった

> AttentionとTransformerは、LLMが文脈を扱いやすくなった理由を理解するための重要な入口です。

## 理解度チェック

Q1. Attentionの説明として最も適切なものはどれですか。

- A. 文章を前から順番に読み、最後の状態だけを使って出力する仕組み
- B. 入力文の中で、出力に関係しやすい部分へ重みを置く仕組み
- C. 入力された文章をすべて同じ重要度として扱う仕組み
- D. モデルの出力を必ず事実に一致させる仕組み

解説: Attentionは、入力内のどの部分を重視するかを扱う仕組みです。

Q2. Transformerの説明として最も適切なものはどれですか。

- A. RNNのように前から順番に処理することだけを目的にした構造
- B. 入力文を1つの固定長情報に圧縮してから翻訳するための辞書
- C. Attentionを中心にして、文中の単語同士の関係を扱いやすくしたモデル構造
- D. 出力の正しさを人間の確認なしで保証する検証機構

解説: Transformerは、Attentionを中心にしたモデル構造で、多くのLLMの土台になっています。

Q3. Attentionが翻訳で役立った理由として最も適切なものはどれですか。

- A. 翻訳文を作るときに、入力文の関連する単語や表現へ注目しやすくなったため
- B. 語順の違う言語でも、入力文をまったく参照せずに出力できるようになったため
- C. RNNの順次処理を強め、最後の単語だけに情報を集めるようになったため
- D. 翻訳結果が必ず正しいことを保証できるようになったため

解説: Attentionにより、翻訳文を作るときに入力文の必要な部分を参照しやすくなり、語順が違う言語でも文脈を扱いやすくなりました。

Q4. TransformerがLLMの次単語予測に役立つ理由として最も適切なものはどれですか。

- A. 直前の1単語だけを見れば、長い文脈を考えなくても十分に予測できるため
- B. Attentionによって文中の関係を見ながら、次に来そうなトークンを予測しやすいため
- C. 予測候補の中から必ず事実として正しい単語だけを選べるため
- D. 学習データを使わずに、毎回その場で文法規則を手書きするため

解説: Transformerは文中の関係を扱いやすく、文脈を見たうえで次に来そうなトークンを予測するLLMの土台になっています。

答え:

- Q1: B
- Q2: C
- Q3: A
- Q4: B
