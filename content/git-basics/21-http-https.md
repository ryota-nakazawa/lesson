# HTTP / HTTPS

HTTPは、ブラウザとサーバーがやりとりするときの通信ルールです。

HTTPSは、HTTPの通信を暗号化して安全性を高めたものです。

> まとめ: HTTP/HTTPSは、ブラウザとサーバーがリクエストとレスポンスをやりとりするためのルールです。

## リクエストとレスポンス

Webでは、ブラウザがサーバーへリクエストを送り、サーバーがレスポンスを返します。

```txt
ブラウザ
  ↓ リクエスト
サーバー
  ↓ レスポンス
ブラウザ
```

たとえば、ページを開く、フォームを送信する、データを取得する、といった操作でHTTP通信が行われます。

## ステータスコード

HTTPでは、レスポンスにステータスコードが含まれます。

よく見る例です。

- `200`: 成功
- `404`: 見つからない
- `500`: サーバー側のエラー

エラー調査では、このステータスコードを見ると原因を絞りやすくなります。

## HTTPSが重要な理由

HTTPSでは通信が暗号化されます。

ログイン情報や個人情報を扱うWebサービスでは、HTTPSが前提になります。

> HTTP/HTTPSを理解すると、API通信やエラー調査の会話がしやすくなります。
