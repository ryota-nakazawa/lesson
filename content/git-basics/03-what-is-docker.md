# Dockerって何？

Dockerは、アプリケーションを動かすために必要な環境をまとめて扱うための仕組みです。

開発では、同じコードでもPCの設定やインストールされているソフトの違いによって、動いたり動かなかったりすることがあります。Dockerを使うと、アプリを動かすための環境をコンテナとしてまとめられます。

> まとめ: Dockerは、アプリを動かす環境をそろえやすくするための道具です。

## なぜDockerを使うのか

チームで開発していると、次のような問題が起きます。

- 自分のPCでは動くが、他の人のPCでは動かない
- 必要なバージョンのNode.jsやDBが違う
- セットアップ手順が長く、環境構築に時間がかかる

Dockerを使うと、実行環境をコードとして管理しやすくなります。

## コンテナとは

コンテナは、アプリケーションを動かすための小さな実行環境です。

```txt
アプリ
必要なライブラリ
実行に必要な設定
  ↓
コンテナとしてまとめる
```

コンテナを使うことで、別のPCでも似た環境を再現しやすくなります。

## Dockerfileとdocker compose

Dockerでは、よく次のようなファイルが出てきます。

- `Dockerfile`: コンテナの作り方を書くファイル
- `compose.yml`: 複数のコンテナをまとめて起動するための設定ファイル

たとえば、Webアプリとデータベースを一緒に起動したいときにDocker Composeを使うことがあります。

```bash
docker compose up
```

このコマンドで、設定に書かれたサービスをまとめて起動できます。

## FDE人材が押さえること

Dockerを完全に書ける必要はなくても、次の点を押さえると実務で会話しやすくなります。

- Dockerは環境差分を減らすために使う
- コンテナはアプリを動かすための実行環境
- `docker compose up` で開発環境を起動することがある
- エラーが出たら、どのコンテナで起きているかを見る

> Dockerは、チームで同じ環境を再現するための土台として使われます。
