# IPアドレス / ポート

Webやアプリ開発では、`localhost:3000` や `127.0.0.1:8000` のような表記を見ることがあります。

これは、**IPアドレス** と **ポート** を組み合わせて、どこに接続するかを表しています。

```txt
127.0.0.1:8000
    ↑      ↑
IPアドレス  ポート
```

> まとめ: IPアドレスは「どのPC・サーバーか」、ポートは「その中のどの入口か」を表します。

たとえるなら、IPアドレスは建物の住所、ポートはその建物の中の入口や部屋番号のようなものです。

```txt
IPアドレス: どの建物かを示す住所
ポート: その建物の中で、どの入口・部屋に行くかを示す番号
```

たとえば、`127.0.0.1:3000` なら、`127.0.0.1` は自分のPCという建物、`3000` はその中でアプリが待っている入口だと考えると分かりやすいです。

## IPアドレスとは

IPアドレスは、ネットワーク上の接続先を識別するための番号です。

たとえば、次のような形式があります。

```txt
192.0.2.10
127.0.0.1
```

ドメイン / DNSのレッスンでは、ドメインがIPアドレスなどの接続先へ変換されると説明しました。

```txt
example.com
  ↓ DNSで名前解決
192.0.2.10
```

IPアドレスは、コンピューターが接続先を見つけるための番号だと考えると分かりやすいです。

## localhostとは

`localhost` は、自分のPCを指す特別な名前です。

開発中のアプリを自分のPCで起動すると、次のようなURLで確認することがあります。

```txt
http://localhost:3000
```

これは、「自分のPCの3000番ポートで動いているアプリにアクセスする」という意味です。

`127.0.0.1` も、自分のPCを指すIPアドレスとしてよく使われます。

```txt
localhost
127.0.0.1
```

初学者のうちは、どちらも「自分のPCを指すもの」と押さえると十分です。

## ポートとは

ポートは、1台のPCやサーバーの中で、どのアプリやサービスに接続するかを分ける番号です。

同じPCの中で、複数のアプリが同時に動くことがあります。

```txt
自分のPC
  ├─ 3000番ポート: フロントエンド開発サーバー
  ├─ 8000番ポート: バックエンドAPI
  └─ 5432番ポート: データベース
```

IPアドレスだけでは「どのPCか」は分かりますが、そのPCの中のどのサービスに接続するかまでは分かりません。

そこで、ポート番号を使って入口を分けます。

## URLで見るIPアドレスとポート

URLでは、ホスト名やIPアドレスの後ろに `:3000` のような形でポート番号を書くことがあります。

| 例 | 意味 |
| --- | --- |
| `http://localhost:3000` | 自分のPCの3000番ポートにアクセス |
| `http://127.0.0.1:8000` | 自分のPCの8000番ポートにアクセス |
| `https://api.example.com` | 通常はHTTPSの標準ポートにアクセス |

開発中は、`3000`、`5173`、`8000` などのポート番号をよく見ます。

一方で、本番のWebサイトでは、URLにポート番号を書かないことも多いです。これは、HTTPやHTTPSでは標準のポート番号があるためです。

| 通信 | 標準ポート |
| --- | --- |
| HTTP | 80 |
| HTTPS | 443 |

たとえば、`https://example.com` は、通常 `443` 番ポートへアクセスしていると考えられます。

つまり、`https://api.example.com` は、実際には `:443` が省略されていると考えると分かりやすいです。

## ポート番号が使われているときのエラー

開発中に、次のようなエラーを見ることがあります。

```txt
Port 3000 is already in use
```

これは、3000番ポートがすでに別のアプリに使われている、という意味です。

この場合は、次のような対応をすることがあります。

- すでに動いているアプリを止める
- 別のポート番号で起動する
- どのプロセスがポートを使っているか確認する

ポートは「入口」なので、同じ入口を複数のアプリが同時に使おうとすると競合が起きます。

## まず押さえること

最初は、次の表記を見たときに何を指しているか分かれば十分です。

| 表記 | ざっくりした意味 |
| --- | --- |
| `localhost` | 自分のPC |
| `127.0.0.1` | 自分のPCを指すIPアドレス |
| `:3000` | 3000番ポート |
| `localhost:3000` | 自分のPCの3000番ポート |

> IPアドレスとポートが分かると、「どこにアクセスしているのか」「どのアプリが待ち受けているのか」を切り分けやすくなります。

## 理解度チェック

Q1. IPアドレスの説明として最も近いものはどれですか。

- A. Webページの見た目を指定するファイル
- B. ネットワーク上の接続先を識別するための番号
- C. Gitのブランチを切り替えるための名前
- D. パッケージのバージョンだけを管理する仕組み

解説: IPアドレスは、ネットワーク上でどのPCやサーバーに接続するかを識別するための番号です。

Q2. `localhost` の説明として最も近いものはどれですか。

- A. 自分のPCを指す特別な名前
- B. 本番環境だけを指すドメイン
- C. 外部のDNSサーバーだけを指す名前
- D. データベースのパスワード

解説: `localhost` は、自分のPCを指す名前です。開発中のアプリを自分のPCで確認するときによく使います。

Q3. `http://localhost:3000` の `3000` が表しているものはどれですか。

- A. Gitのコミット番号
- B. HTTPのステータスコード
- C. 自分のPCの中の接続先を分けるポート番号
- D. DNSレコードの種類

解説: `:3000` はポート番号です。自分のPCの中で、どのアプリやサービスに接続するかを分ける入口のようなものです。

Q4. `Port 3000 is already in use` と表示されたときの意味として最も近いものはどれですか。

- A. 3000番ポートがまだ一度も使われていない
- B. 3000番ポートはHTTPSの標準ポートである
- C. DNSの設定が必ず削除された
- D. 3000番ポートがすでに別のアプリに使われている

解説: このエラーは、指定したポートがすでに別のアプリに使われていて、同じ入口を使えない状態を表します。

Q5. `localhost:3000` は開けるのに、`localhost:8000` は開けない場合にまず考えることとして最も近いものはどれですか。

- A. 8000番ポートで待ち受けているアプリやAPIが起動していない可能性がある
- B. 3000番ポートは必ず本番環境を指す
- C. `localhost` は外部の本番サーバーだけを指す
- D. DNSレコードが必ず削除されている

解説: `localhost:8000` にアクセスできない場合、まずは8000番ポートで対象のアプリやAPIが起動しているかを確認します。

答え:

- Q1: B
- Q2: A
- Q3: C
- Q4: D
- Q5: A
