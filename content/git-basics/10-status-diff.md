# 状態を確認する: status / diff

Gitで作業するときは、まず状態を確認する習慣が大切です。

状態を確認する代表的なコマンドが `git status` と `git diff` です。

> まとめ: 迷ったら、まず `git status` で今の状態を確認します。

## git status

`git status` は、リポジトリの現在の状態を確認するコマンドです。

```bash
git status
```

このコマンドを使うと、次のようなことが分かります。

- どのブランチにいるか
- 変更されたファイルがあるか
- まだ記録対象にしていない変更があるか
- 次のコミットに入る予定の変更があるか

たとえば、次のように表示されます。

```bash
$ git status
On branch main
Your branch is up to date with 'origin/main'.

Changes not staged for commit:
  modified:   README.md

Untracked files:
  memo.md

no changes added to commit
```

この例では、次のことが分かります。

- 今いるブランチは `main`
- `README.md` が変更されている
- `memo.md` はまだGitが管理していない新しいファイル
- まだ次のコミットに入る準備はできていない

Gitの操作に慣れるまでは、何かする前後に `git status` を見るだけでも安心して作業しやすくなります。

> 補足: 今いるブランチは `git status` でも分かります。ブランチ一覧を見たいときは `git branch` を使います。

## modified と staged

`git status` では、変更の状態を表す言葉が出てきます。

- `modified`: ファイルが変更されている
- `staged`: 次のコミットに入る準備ができている
- `untracked`: Gitがまだ管理していない新しいファイル

最初はすべてを完璧に覚える必要はありません。まずは「変更されているファイル」と「次のコミットに入るファイル」が分かれていることを押さえます。

## git diff

`git diff` は、変更内容の差分を確認するコマンドです。

```bash
git diff
```

`git status` が「どのファイルが変わったか」を見るためのものだとすると、`git diff` は「中身がどう変わったか」を見るためのものです。

たとえば、READMEの文章を変えた場合、次のように表示されます。

```diff
$ git diff
diff --git a/README.md b/README.md
--- a/README.md
+++ b/README.md
@@
-このツールは社内メモを管理します。
+このツールはチームの学習コンテンツを管理します。
```

差分では、行の先頭に記号が付きます。

- `-`: 削除された行
- `+`: 追加された行

この例では、「社内メモを管理します」という説明が削除され、「チームの学習コンテンツを管理します」という説明が追加されたことが分かります。

## 共有前に差分を見る

変更をコミットしたり、リモートに送ったりする前に差分を見ると、意図しない変更に気づきやすくなります。

- 不要なログを入れたままにしていないか
- 関係ないファイルを変更していないか
- 文章やコードに余計な修正が混ざっていないか

> `status` で全体を見て、`diff` で中身を見る。この2つはGit作業の基本動作です。

## 理解度チェック

Q1. `git status` で確認できることとして最も近いものはどれですか。

- A. GitHubのパスワード
- B. 今いるブランチや変更されたファイルの状態
- C. 画像ファイルの解像度
- D. パソコンの空き容量

解説: `git status` では、今いるブランチ、変更されたファイル、ステージされている変更などを確認できます。

Q2. `untracked` の説明として最も近いものはどれですか。

- A. すでにコミット済みのファイル
- B. 次のコミットに入る準備ができているファイル
- C. Gitがまだ管理していない新しいファイル
- D. リモートから削除されたファイル

解説: `untracked` は、Gitがまだ管理対象として追跡していない新しいファイルを表します。

Q3. `git diff` の役割として最も近いものはどれですか。

- A. 変更内容の差分を確認する
- B. ブランチを新しく作る
- C. 変更をリモートへ送る
- D. コミットメッセージを保存する

解説: `git diff` は、ファイルの中身がどのように変わったかを確認するためのコマンドです。

Q4. 差分表示で `+` が付いている行は何を表しますか。

- A. 削除された行
- B. 変更してはいけない行
- C. Gitが無視する行
- D. 追加された行

解説: 差分表示では、`+` が追加された行、`-` が削除された行を表します。

答え:

- Q1: B
- Q2: C
- Q3: A
- Q4: D
