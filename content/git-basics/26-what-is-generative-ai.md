# 生成AIとは何か

生成AIは、文章、画像、音声、コードなどを新しく作り出すAIです。

従来のAIが「分類する」「予測する」ことを得意としていたのに対して、生成AIは入力された指示や文脈をもとに、新しい出力を作ることができます。

```txt
指示や入力
  ↓
生成AI
  ↓
文章・画像・コードなどを生成
```

> まとめ: 生成AIは、学習したパターンをもとに新しいコンテンツを作るAIです。

## 従来のAIと生成AIの違い

従来のAIは、入力された情報をもとに「判定する」「分類する」「予測する」ことを得意としていました。

一方で生成AIは、入力された指示や文脈をもとに、文章、画像、コードなどの新しい出力を作ることを得意とします。

| 種類 | 得意なこと | 例 |
| --- | --- | --- |
| 従来のAI | 判定、分類、予測 | 迷惑メールか判定する、売上を予測する、画像に猫が写っているか分類する |
| 生成AI | 新しい出力を作る | メール文を作る、コードを書く、説明文を要約する、画像を生成する |

たとえば、従来のAIは「この問い合わせは請求カテゴリか、技術カテゴリか」を分類するのに向いています。

生成AIは「この問い合わせへの返信文を作る」「問い合わせ内容を要約する」「対応手順の案を出す」のように、次に使える文章やアイデアを作るのに向いています。

```txt
従来のAI: 入力を見て、分類・判定・予測する
生成AI: 入力を見て、新しい文章・画像・コードなどを作る
```

どちらが優れているというより、目的が違います。実務では、分類や予測には従来型のAI、文章作成やアイデア出しには生成AI、というように使い分けます。

## 何を生成できるか

生成AIが作れるものは、文章だけではありません。

| 種類 | 例 |
| --- | --- |
| 文章 | 要約、メール文、記事、説明文 |
| コード | 関数、テスト、修正案 |
| 画像 | イラスト、写真風画像、図解 |
| 音声 | 読み上げ、音声合成 |
| 動画 | 短い映像、アニメーション |

実務では、文章作成、調査補助、コード生成、画像生成、問い合わせ対応など、幅広い場面で使われます。

## 生成AIとLLM

LLMは、Large Language Modelの略です。日本語では大規模言語モデルと呼ばれます。

LLMは、文章を扱う生成AIの中心的な技術です。

| 言葉 | 意味 |
| --- | --- |
| 生成AI | 文章、画像、音声、コードなどを生成するAI全般 |
| LLM | 文章やコードなどの言語を扱う大規模なモデル |

ChatGPTのような対話AIは、LLMを使って文章を理解し、回答を生成します。

## 生成AIはなぜ便利か

生成AIは、ゼロから考える作業を補助できます。

たとえば、次のような使い方があります。

- 文章のたたき台を作る
- 難しい内容をやさしく説明する
- コードの例を出す
- エラーの原因を整理する
- アイデアを複数出す

生成AIは「答えを丸投げする道具」というより、考える速度を上げる補助役として使うと効果的です。

## 注意点

生成AIの出力は、常に正しいとは限りません。

次のような注意が必要です。

| 注意点 | 内容 |
| --- | --- |
| 誤情報 | もっともらしい誤りを出すことがある |
| 最新情報 | 学習時点や接続先によって古い情報になることがある |
| 著作権・ライセンス | 生成物や参照元の扱いに注意が必要 |
| 秘密情報 | 機密情報や個人情報を不用意に入力しない |

生成AIを使うときは、出力をそのまま信じるのではなく、人間が確認することが大切です。

## まず押さえること

- 生成AIは、文章や画像などを新しく作るAI
- 従来のAIは分類や予測、生成AIは新しい出力を作ることが得意
- LLMは、言語を扱う生成AIの中心的な技術
- 生成AIは考える速度を上げる補助役になる
- 出力は必ず確認し、誤情報や秘密情報に注意する

> 生成AIは、正しく使えば強力な補助になります。ただし、最後に判断するのは人間です。
