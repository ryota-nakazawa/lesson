# ドメイン / DNS

ドメインは、Webサイトの住所のようなものです。

たとえば、`example.com` のような文字列がドメインです。

DNSは、そのドメインを実際の接続先へ変換する仕組みです。

> まとめ: ドメインは人間が読みやすい住所、DNSはその住所を接続先へ変換する仕組みです。

## なぜDNSが必要か

コンピューター同士は、最終的にはIPアドレスのような接続先情報を使って通信します。

しかし、人間が毎回IPアドレスを覚えるのは大変です。

そこで、`example.com` のようなドメインを使います。

```txt
example.com
  ↓ DNSで変換
接続先のサーバー
```

## よく見る言葉

- ドメイン: `example.com` のような名前
- サブドメイン: `app.example.com` のような名前
- DNSレコード: ドメインをどこへ向けるかの設定

## FDE人材が押さえること

Webサイトを公開するときは、アプリをデプロイするだけでなく、ドメインが正しい接続先を向いている必要があります。

> ドメインとDNSが分かると、「URLでアクセスできるようにする」流れが理解しやすくなります。
