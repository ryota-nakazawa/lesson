# クラウド / インフラ

インフラは、アプリケーションを動かすための土台です。

クラウドは、その土台をインターネット経由で利用できるサービスです。

> まとめ: インフラはアプリを支える土台、クラウドはその土台を必要に応じて使える仕組みです。

## インフラに含まれるもの

Webアプリを動かすには、コードだけでなく周辺の土台が必要です。

たとえば、次のようなものがあります。

- サーバー
- ネットワーク
- データベース
- ストレージ
- ログや監視
- セキュリティ設定

## クラウドとは

クラウドを使うと、自分で物理サーバーを用意しなくても、サーバーやデータベースなどを利用できます。

```txt
アプリ
  ↓
クラウド上のサーバー / DB / ストレージ
```

必要に応じて増やしたり、設定を変えたりしやすい点が特徴です。

## FDE人材が押さえること

細かいクラウド設定をすべて覚える必要はありません。

まずは、アプリが動くには次のような土台が必要だと理解します。

- どこでアプリが動くか
- どこにデータを保存するか
- どうやってアクセスするか
- 問題が起きたときにどこを見るか

> インフラを理解すると、デプロイ後の動作や障害調査の会話がしやすくなります。
