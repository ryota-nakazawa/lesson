# MCPって何？

MCPは、Model Context Protocolの略です。

AIエージェントが外部のツールやデータに接続するための共通ルールとして使われます。

> まとめ: MCPは、AIと外部ツールをつなぐための接続ルールです。

## なぜMCPが必要か

AIにできることは、会話だけではありません。

外部ツールとつながることで、ファイルを読んだり、チケットを確認したり、データベースの情報を見たりできます。

ただし、ツールごとに接続方法がバラバラだと扱いにくくなります。

MCPは、その接続方法をそろえるための仕組みです。

```txt
AIエージェント
  ↓ MCP
外部ツール / データ / サービス
```

## MCPサーバーとクライアント

MCPでは、ツールやデータを提供する側をMCPサーバーと呼びます。

それを利用するAIエージェント側がクライアントです。

```txt
MCPクライアント: AIエージェント側
MCPサーバー: ツールやデータを提供する側
```

## FDE人材が押さえること

MCPを理解すると、AIがなぜGitHub、Slack、Google Drive、ローカルファイルなどと連携できるのかを捉えやすくなります。

> MCPは、AIエージェントが外部世界と安全にやりとりするための橋渡しとして理解すると分かりやすいです。
