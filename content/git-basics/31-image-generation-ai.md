# 画像生成AIの仕組み

画像生成AIは、文章の指示から画像を作るAIです。

「猫のイラストを作って」「Webサービスの図解を作って」のようなプロンプトを入力すると、学習したパターンをもとに画像を生成します。

> まとめ: 画像生成AIは、画像を数値として扱い、プロンプトに合う画像へ少しずつ近づけて生成します。

## 画像は数値として扱われる

コンピュータにとって、画像はピクセルの集まりです。

それぞれのピクセルには、色を表す数値があります。

```txt
画像
  ↓
ピクセルの集まり
  ↓
色を表す数値
```

つまり画像生成AIは、人間のように「絵」を直接見ているというより、画像を数値のパターンとして扱っています。

## 画像空間と潜在空間

画像はそのままだと非常に大きなデータです。

そこで、画像の特徴を扱いやすい数値空間に圧縮して考えることがあります。このような表現を、潜在空間と呼ぶことがあります。

| 言葉 | イメージ |
| --- | --- |
| 画像空間 | ピクセルとして表された画像そのものの世界 |
| 潜在空間 | 画像の特徴を圧縮して表した数値の世界 |

初学者のうちは、潜在空間を「画像の特徴を整理した地図」のように考えると分かりやすいです。

近い場所には似た雰囲気の画像があり、遠い場所には違う特徴の画像がある、というイメージです。

たとえば、猫の画像に近い場所、車の画像に近い場所、写真風の画像に近い場所、イラスト風の画像に近い場所があると考えるとイメージしやすくなります。

```txt
潜在空間のイメージ

猫っぽい場所       車っぽい場所
    ○                 ○

写真風の場所       イラスト風の場所
    ○                 ○
```

この空間は実際には2次元の地図ではなく、多数の数値で表される高次元の空間です。

球体のようにイメージするのは、直感をつかむための比喩としては使えます。ただし、実際に「外側から中心へ向かう」というより、ノイズの多い点から、プロンプトに合う画像らしい点へ少しずつ移動するイメージの方が近いです。

## 拡散モデルの考え方

画像生成AIでは、拡散モデルという考え方がよく使われます。

大まかには、ノイズから少しずつ画像らしいものへ戻していく仕組みです。

```txt
ノイズだらけの画像
  ↓ 少しノイズを取り除く
ぼんやりした形
  ↓ さらに整える
プロンプトに合う画像
```

学習時には、画像にノイズを加える過程と、そのノイズを取り除く過程を学びます。

生成時には、ランダムなノイズから始めて、プロンプトに合う方向へ少しずつ整えていきます。

## 潜在空間と拡散モデルの関係

潜在空間と拡散モデルは、別々の話ではありません。

多くの画像生成AIでは、画像そのもののピクセル空間で直接すべてを処理するのではなく、画像の特徴を圧縮した潜在空間上でノイズを取り除いていきます。

大まかには、次のような流れです。

```txt
ランダムなノイズ
  ↓
潜在空間上のぼんやりした点
  ↓ プロンプトに合う方向へ少しずつノイズを除去
画像らしい特徴を持つ点
  ↓
ピクセルの画像として出力
```

たとえば「白い猫、アイコン風、シンプルな背景」と入力した場合、モデルはただランダムにノイズを消すのではありません。

プロンプトの情報を手がかりにしながら、潜在空間の中で「白い猫」「アイコン風」「シンプルな背景」に近い特徴を持つ方向へ、少しずつ整えていきます。

このときの「近づく」は、1枚の完成画像へ一直線に向かうというより、ノイズの多い状態から、プロンプトと整合する特徴が強い状態へ何段階もかけて移っていくイメージです。

| 観点 | 役割 |
| --- | --- |
| 潜在空間 | 画像の特徴を扱いやすく圧縮した数値空間 |
| 拡散モデル | ノイズを少しずつ取り除き、画像らしい特徴へ整える仕組み |
| プロンプト | どの方向へ整えるかを指定する手がかり |

## プロンプトが画像を導く

画像生成では、プロンプトが方向づけになります。

たとえば次のような違いがあります。

| プロンプト | 生成されやすい画像 |
| --- | --- |
| 猫 | 猫の画像 |
| 白い猫、シンプルな背景 | 白い猫と余白のある画像 |
| 白い猫、アイコン風、フラットデザイン | アイコンとして使いやすい画像 |

プロンプトが具体的になるほど、生成される画像の方向性も指定しやすくなります。

ただし、細部まで常に正確に制御できるわけではありません。

## 文字を含む画像も生成できる理由

最近の画像生成AIは、看板、ポスター、UI、資料、漫画の吹き出しなど、文字を含む画像も以前より自然に生成できるようになっています。

これは、文字そのものを特別扱いしているというより、文字を含む画像の特徴も画像データの一部として学習しているためです。

たとえば、学習データには次のような情報が含まれます。

| 学習される情報 | 例 |
| --- | --- |
| 文字の形 | アルファベット、日本語、数字の見た目 |
| 文字の配置 | 看板の中央、ボタンの中、ポスターの見出し |
| 背景との関係 | 白背景に黒文字、写真の上に大きな文字 |
| プロンプトとの対応 | `SALE` と書かれたポスター、`CAFE` の看板 |

つまり、文字を含む画像も、潜在空間の中では「文字がある画像らしさ」「ポスターらしさ」「UIらしさ」のような特徴として表現されます。

```txt
プロンプト: "SALE と書かれた赤いポスター"
  ↓
文字を含むポスター画像の特徴へ誘導
  ↓
ノイズを少しずつ除去
  ↓
文字とレイアウトを含む画像
```

ただし、画像生成AIにとって文字は、まずは画像の模様として扱われます。

そのため、最近のモデルでは文字の再現性がかなり上がっている一方で、細かい綴り、長い文章、日本語の字形、正確なレイアウトは崩れることがあります。

重要な文言を含む画像を使う場合は、生成後に文字を確認し、必要ならデザインツールで文字だけ入れ直す方が安全です。

## 注意点

画像生成AIを使うときは、次の点に注意します。

| 注意点 | 内容 |
| --- | --- |
| 権利 | 既存作品や人物に似せすぎない |
| 正確性 | 文字、細部、構造が崩れることがある |
| 用途 | 商用利用や公開時は利用規約を確認する |
| 偏り | 学習データ由来の偏りが出ることがある |

図解やラフ案を作る用途では便利ですが、最終成果物として使う場合は確認が必要です。

## まず押さえること

- 画像生成AIは、画像を数値のパターンとして扱う
- 潜在空間は、画像の特徴を圧縮して扱うための考え方
- 拡散モデルは、潜在空間上のノイズをプロンプトに合う方向へ少しずつ整える仕組み
- 文字を含む画像も、文字の形や配置を含む視覚特徴として学習される
- プロンプトで方向づけできるが、細部の確認は必要

> 画像生成AIは、アイデア出しや図解作成に強力ですが、権利や正確性の確認まで含めて使うことが大切です。

## 理解度チェック

Q1. 画像生成AIが画像を扱う考え方として最も適切なものはどれですか。

- A. 画像を人間と同じ意味理解だけで扱い、数値としては扱わない
- B. 画像をピクセルや特徴量などの数値パターンとして扱う
- C. 画像をファイル名だけで判断し、中身の特徴は使わない
- D. 画像を常に2次元の地図としてだけ保存する

解説: コンピュータは画像をピクセルや特徴量などの数値として扱います。

Q2. 潜在空間の説明として最も適切なものはどれですか。

- A. 画像の特徴を扱いやすく圧縮して表した高次元の数値空間
- B. 画像を必ず球体の外側から中心へ移動させるための2次元平面
- C. 生成後の画像ファイルを保存するフォルダ
- D. プロンプトを使わずに毎回同じ画像を出す仕組み

解説: 潜在空間は、画像などの特徴を扱いやすい数値表現にした空間として考えられます。

Q3. 潜在空間と拡散モデルの関係として最も適切なものはどれですか。

- A. 潜在空間は完成画像だけを保存し、拡散モデルはノイズを増やし続ける
- B. 拡散モデルはプロンプトと関係なく、毎回同じ画像へ近づける
- C. 潜在空間は必ず2次元の地図で、拡散モデルはプロンプトを使わない
- D. 潜在空間上のノイズを、プロンプトに合う画像らしい特徴へ少しずつ整える

解説: 拡散モデルでは、潜在空間上のノイズをプロンプトに合う方向へ少しずつ整え、画像らしい特徴へ近づけていきます。

Q4. 文字を含む画像生成についての説明として最も適切なものはどれですか。

- A. 文字は画像とは別のDBに保存されるため、潜在空間とは関係しない
- B. 文字を含む画像の形、配置、背景との関係も視覚特徴として学習される
- C. 画像生成AIは文字を完全に理解しているため、長文でも必ず正確に描ける
- D. プロンプト内の文字列は画像生成に一切影響しない

解説: 文字を含む画像も、文字の形や配置、背景との関係などを含む視覚特徴として学習されます。ただし、文字の正確性は確認が必要です。

Q5. 画像生成AIを使うときの注意点として適切なものはどれですか。

- A. 出力画像は必ず権利確認不要で使える
- B. 文字や細部は必ず正確なので、公開前の確認は不要である
- C. 商用利用や公開時は利用規約、権利面、文字の正確性を確認する
- D. プロンプトが具体的であれば、権利やライセンスの確認は不要になる

解説: 画像生成AIの出力は、用途に応じて権利、利用規約、細部の正確性を確認する必要があります。

答え:

- Q1: B
- Q2: A
- Q3: D
- Q4: B
- Q5: C
