# CLI/コマンドって何？

CLIは、**Command Line Interface** の略です。日本語では「コマンドラインインターフェース」と呼ばれます。

簡単にいうと、文字でコンピューターに指示を出すための操作方法です。

普段のPC操作では、ボタンやメニューをクリックして操作することが多いです。このような画面操作はGUIと呼ばれます。

一方で、CLIではターミナルやコマンドプロンプトに文字を入力して操作します。

```bash
git status
```

このように、やってほしい操作を文字で入力するものがコマンドです。

> まとめ: CLIは文字で操作する方法、コマンドはコンピューターに出す具体的な指示です。

## GUIとCLIの違い

GUIは、画面上のボタンやメニューを使って操作します。

CLIは、文字で命令を入力して操作します。

```txt
GUI: ボタンをクリックして操作する
CLI: コマンドを入力して操作する
```

たとえば、ファイル一覧を見る操作でも、GUIではFinderやエクスプローラーを開きます。CLIでは、次のようなコマンドを入力します。

```bash
ls
```

CLIは最初はとっつきにくく見えますが、慣れると同じ操作を正確に繰り返しやすくなります。

## なぜCLIを使うのか

開発ではCLIをよく使います。

理由は、開発ツールの多くがコマンドで操作できるように作られているからです。

たとえば、Gitでは状態確認やコミットをコマンドで行います。

```bash
git status
git add README.md
git commit -m "READMEの説明を更新"
```

Node.jsやnpm、DockerなどもCLIで操作することが多いです。

```bash
npm install
npm run dev
docker compose up
```

CLIを使えるようになると、次のようなことがしやすくなります。

- 開発ツールを操作する
- 作業を手順として共有する
- 同じ操作を何度も再現する
- エラー内容を確認して相談する

> CLIは、開発者だけの特殊技能ではありません。チームで開発ツールを扱うための共通の操作方法です。

## 生成AI時代にCLIが便利な理由

生成AIを使って開発を進めるときも、CLIはとても相性がよい操作方法です。

理由は、コマンドと実行結果をそのまま共有できるからです。

たとえば、エラーが出たときに、次のように伝えられます。

```txt
実行したコマンド:
npm run dev

表示されたエラー:
Error: Cannot find module ...
```

GUIの画面操作だけだと、「どこを押したか」「何が起きたか」を言葉で説明する必要があります。一方でCLIでは、実行したコマンドと出力された結果をそのままコピーして共有できます。

生成AIに相談するときも、次の情報があると原因を絞り込みやすくなります。

- 実行したコマンド
- 実行したフォルダ
- 表示されたエラー
- 直前に何を変更したか

また、生成AIに「次に実行するコマンド」を提案してもらい、その結果をまた共有する、という進め方もしやすくなります。

```txt
人間: このエラーの原因を見たい
AI: まず `pwd` と `ls` と `npm run dev` の結果を見せてください
人間: コマンド結果を貼る
AI: 結果を見て次の修正を提案する
```

> CLIを使えると、生成AIとのやりとりが「なんとなく動かない」から「このコマンドでこの結果になった」に変わります。

## ターミナルとは何か

ターミナルは、CLIでコマンドを入力するためのアプリです。

Macでは「ターミナル」、Windowsでは「PowerShell」や「コマンドプロンプト」、開発環境ではVS Codeのターミナルを使うことがあります。

ターミナルを開くと、次のような入力待ちの状態になります。

```bash
$
```

この `$` のような記号はプロンプトと呼ばれます。ここにコマンドを入力して実行します。

```bash
$ git status
```

資料や記事では、入力するコマンドの前に `$` が書かれていることがあります。その場合、`$` は入力しません。

```bash
# 入力するのはこれ
git status

# $ は入力しない
$ git status
```

## コマンドの基本形

コマンドは、だいたい次のような形で書かれます。

```txt
コマンド名 サブコマンド 対象 オプション
```

Gitの例で見ると、次のようになります。

```bash
git commit -m "READMEの説明を更新"
```

このコマンドは、次の部品に分けられます。

- `git`: コマンド名
- `commit`: Gitに何をさせるか
- `-m`: メッセージを指定するオプション
- `"READMEの説明を更新"`: コミットメッセージ

別の例も見てみます。

```bash
git add README.md
```

この場合は、`git` に対して、`README.md` を `add` するように指示しています。

## オプションと引数

CLIでは、コマンドの後ろに追加情報を付けることがあります。

対象を表すものを引数、動き方を変えるものをオプションと考えると分かりやすいです。

```bash
git add README.md
```

この例では、`README.md` が引数です。どのファイルを対象にするかを指定しています。

```bash
git commit -m "READMEの説明を更新"
```

この例では、`-m` がオプションです。コミットメッセージをコマンド内で指定するために使っています。

オプションには、短い書き方と長い書き方があります。

```bash
git commit -m "説明を更新"
```

```bash
npm install --save-dev vite
```

最初からすべて覚える必要はありません。まずは「コマンド名の後ろに、対象や設定を付けることがある」と理解できれば十分です。

## よく使う基本コマンド

CLIに慣れるために、まずは次のコマンドを知っておくと便利です。

CLIでは、「今どのフォルダにいるか」が重要です。コマンドは基本的に現在いるフォルダを基準に動くため、まず `pwd`、`ls`、`cd` の3つに慣れると理解しやすくなります。

### pwd

```bash
$ pwd
/Users/user/Desktop/team-project
```

`pwd` は、今いるフォルダを表示するコマンドです。

この例では、現在地が `/Users/user/Desktop/team-project` であることが分かります。

### ls

```bash
$ ls
README.md
package.json
src
```

`ls` は、今いるフォルダの中身を表示するコマンドです。

この例では、`README.md`、`package.json`、`src` があることが分かります。

### cd

```bash
$ cd src
$ pwd
/Users/user/Desktop/team-project/src
```

`cd` は、指定したフォルダへ移動するコマンドです。

この例では、`src` フォルダへ移動しています。`cd` は移動するだけなので、成功しても何も表示されないことがあります。その場合は、`pwd` で移動後の場所を確認できます。

### mkdir

```bash
$ mkdir docs
$ ls
README.md
docs
package.json
src
```

`mkdir` は、新しいフォルダを作るコマンドです。

この例では、`docs` フォルダを作り、その後 `ls` で作成されたことを確認しています。

### clear

```bash
$ clear
```

`clear` は、ターミナルの表示を整理するコマンドです。

実行すると、画面上の表示が消えて、入力しやすい状態になります。ファイルや履歴を削除するコマンドではありません。

> Windowsの環境や設定によっては、コマンド名や表示が少し違うことがあります。まずは自分が使う環境でよく使うものから覚えれば十分です。

## コマンドを使うときの注意点

CLIは便利ですが、入力した内容がそのまま実行されます。

特に、ファイルを削除するコマンドや、広い範囲に影響するコマンドは注意が必要です。

初学者のうちは、次の点を意識します。

- どのフォルダで実行しているか確認する
- コピーしたコマンドをそのまま実行する前に意味を読む
- エラーが出たら、焦らずメッセージを読む
- 危なそうなコマンドは実行前に相談する

エラーが出た場合は、エラーメッセージをそのまま共有すると、相談しやすくなります。

```txt
どのコマンドを実行したか
どのフォルダで実行したか
どんなエラーが出たか
```

この3つが分かると、原因を追いやすくなります。

## まず押さえること

CLIを最初から完璧に覚える必要はありません。

まずは、次の4つを押さえます。

- CLIは文字で操作する方法
- コマンドはコンピューターへの指示
- コマンドは現在いるフォルダを基準に動く
- 開発ではGit、npm、DockerなどをCLIで操作することが多い

> CLIは、開発作業を進めるための操作画面です。よく使うコマンドから少しずつ慣れていけば十分です。

## 理解度チェック

Q1. CLIの説明として最も近いものはどれですか。

- A. 画像編集ソフトの名前
- B. Command Line Interfaceの略で、文字でコンピューターに指示を出す操作方法
- C. Webサイトのデザインだけを作る仕組み
- D. GitHubのレビュー機能

解説: CLIはCommand Line Interfaceの略で、ターミナルなどにコマンドを入力して操作する方法です。

Q2. 生成AI時代にCLIが便利な理由として本文に近いものはどれですか。

- A. コマンドと実行結果をそのまま共有でき、エラー相談や原因調査がしやすいから
- B. 画面操作がすべて不要になり、確認しなくても必ず成功するから
- C. 生成AIがファイルを自動で削除してくれるから
- D. CLIを使うとレビューが不要になるから

解説: CLIでは、実行したコマンドと表示された結果をそのまま共有できるため、生成AIにも状況を伝えやすくなります。

Q3. 今いるフォルダを確認するコマンドはどれですか。

- A. `ls`
- B. `cd`
- C. `pwd`
- D. `clear`

解説: `pwd` は、現在いるフォルダを表示するコマンドです。

Q4. コマンドを使うときの注意点として最も近いものはどれですか。

- A. どんなコマンドでも意味を確認せずに実行する
- B. コマンドは必ずGitHub上でだけ実行する
- C. エラーが出たらメッセージを読まずに閉じる
- D. どのフォルダで実行しているか確認し、意味を読んでから実行する

解説: CLIでは入力した内容がそのまま実行されます。現在地やコマンドの意味を確認してから実行することが大切です。

- Q1: B
- Q2: A
