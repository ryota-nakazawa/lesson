# ローカル環境と本番環境

ローカル環境は、自分のPC上で開発や確認をするための環境です。

本番環境は、実際の利用者がアクセスする環境です。

```txt
ローカル環境: 自分のPCで確認する
本番環境: 利用者が使う
```

> まとめ: ローカルで動くことと、本番で安全に動くことは別です。

## ローカル環境

ローカル環境では、開発中のアプリを自分のPCで動かします。

```bash
npm run dev
```

このようなコマンドでローカルサーバーを立て、`localhost` で確認することがあります。

## 本番環境

本番環境は、利用者が実際に使う環境です。

本番環境では、次のような点が重要になります。

- 利用者に影響する
- データを安全に扱う必要がある
- エラーや停止を避ける必要がある
- 設定や権限を慎重に扱う必要がある

## 環境が分かれる理由

開発中の変更をいきなり本番に出すと、利用者に不具合が見えてしまうことがあります。

そのため、ローカル環境、プレビュー環境、ステージング環境、本番環境のように分けて確認することがあります。

```txt
ローカル → プレビュー / ステージング → 本番
```

> 環境を分けることで、確認中の変更と利用者向けの安定した状態を分けて扱えます。
