# DB / ストレージ

DBはデータベースの略で、構造化されたデータを保存・検索するための仕組みです。

ストレージは、ファイルや画像などのデータを保存する場所として使われます。

> まとめ: DBは検索・更新しやすいデータの置き場、ストレージはファイルの置き場として考えると分かりやすいです。

## データベースとは

データベースには、ユーザー情報、注文情報、記事、設定などを保存します。

```txt
users
orders
articles
settings
```

バックエンドは、データベースから必要な情報を取得したり、更新したりします。

## ストレージとは

ストレージには、画像、PDF、動画、アップロードファイルなどを保存することがあります。

たとえば、プロフィール画像や添付ファイルはストレージに置かれることがあります。

## 使い分け

```txt
DB: 検索・集計・更新したいデータ
ストレージ: ファイルとして保存したいデータ
```

実際のサービスでは、DBにファイルのURLやメタ情報を保存し、ファイル本体はストレージに置くことがあります。

> DBとストレージの違いが分かると、データがどこに保存されているかを考えやすくなります。
