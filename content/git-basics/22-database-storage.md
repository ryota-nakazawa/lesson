# DB / ストレージ

DBはデータベースの略で、構造化されたデータを保存・検索・更新するための仕組みです。

ストレージは、画像、PDF、動画などのファイルを保存する場所として使われます。

```txt
DB: ユーザー情報、注文情報、記事、設定など
ストレージ: 画像、PDF、動画、アップロードファイルなど
```

> まとめ: DBは検索・更新しやすいデータの置き場、ストレージはファイルの置き場として考えると分かりやすいです。

## データベースとは

データベースには、アプリで扱うデータを整理して保存します。

たとえば、次のようなデータです。

| データ | 例 |
| --- | --- |
| ユーザー情報 | 名前、メールアドレス、ログイン情報 |
| 注文情報 | 商品、金額、購入日時 |
| 記事 | タイトル、本文、公開状態 |
| 設定 | 通知設定、表示設定 |

多くのWebアプリでは、バックエンドがデータベースに問い合わせて、必要な情報を取得・保存します。

```txt
フロントエンド
  ↓ APIリクエスト
バックエンド
  ↓ DBへ問い合わせ
データベース
```

## DBMSとSQL

実務では、PostgreSQL、MySQL、SQLiteのようなデータベース管理システムを使うことがあります。

このような仕組みは、アプリとデータの間に入って、データの保存・検索・更新を担当します。ミドルウェアの一種として扱われることもあります。

| 言葉 | 意味 |
| --- | --- |
| DB | データを保存する場所や仕組み |
| DBMS | データベースを管理するソフトウェア |
| SQL | データベースに問い合わせるための言語 |

SQLは、データベースからデータを取得したり、追加・更新・削除したりするために使われる言語です。

## SQLでデータを取得する例

たとえば、ユーザー情報を保存する `users` テーブルがあるとします。

| id | email | name |
| --- | --- | --- |
| 1 | `taro@example.com` | Taro |
| 2 | `hanako@example.com` | Hanako |

すべてのユーザーを取得するSQLは、次のように書けます。

```sql
SELECT * FROM users;
```

特定のメールアドレスを持つユーザーだけを取得したい場合は、条件を付けます。

```sql
SELECT * FROM users
WHERE email = 'taro@example.com';
```

このSQLは、「`users` テーブルから、`email` が `taro@example.com` の行を探してください」という意味です。

```txt
SQLを書く
  ↓
DBMSがデータベースを検索する
  ↓
条件に合うデータが返る
```

## ログイン情報の例

ログイン機能では、ユーザーが入力したメールアドレスやパスワードをもとに、本人かどうかを確認します。

ただし、実務ではパスワードをそのままDBに保存しません。通常は、ハッシュ化されたパスワードを保存します。

たとえば、DBには次のような形で保存されることがあります。

| id | email | password_hash |
| --- | --- | --- |
| 1 | `taro@example.com` | `hashed_password_value` |

ログイン時の大まかな流れは次の通りです。

```txt
ユーザーがメールアドレスとパスワードを入力
  ↓
バックエンドが email でユーザーを検索
  ↓
入力されたパスワードと保存済みの password_hash を照合
  ↓
一致すればログイン成功
```

メールアドレスでユーザーを探すSQLのイメージは次のようになります。

```sql
SELECT id, email, password_hash
FROM users
WHERE email = 'taro@example.com';
```

> パスワードをそのままDBに保存するのは危険です。ここでは、ログイン時にDBを検索する流れを理解することが目的です。

## CRUDとは

CRUDは、データを扱う基本操作をまとめた言葉です。

| 操作 | 意味 | SQLの例 |
| --- | --- | --- |
| Create | 新しく作る | `INSERT` |
| Read | 読み取る | `SELECT` |
| Update | 更新する | `UPDATE` |
| Delete | 削除する | `DELETE` |

Webアプリの多くの機能は、CRUDの組み合わせで考えられます。

| 機能 | CRUDで見ると |
| --- | --- |
| 新しい記事を投稿する | Create |
| 記事一覧を見る | Read |
| プロフィールを編集する | Update |
| 投稿を削除する | Delete |

たとえば、ユーザー名を更新するSQLは次のようなイメージです。

```sql
UPDATE users
SET name = 'New Name'
WHERE id = 1;
```

ユーザーを削除するSQLは次のようなイメージです。

```sql
DELETE FROM users
WHERE id = 1;
```

実務では、安全性や権限確認を行ったうえで操作します。特に `UPDATE` や `DELETE` は、条件を間違えると意図しないデータまで変更・削除してしまうため注意が必要です。

## ストレージとは

ストレージには、画像、PDF、動画、アップロードファイルなどを保存することがあります。

たとえば、プロフィール画像や添付ファイルはストレージに置かれることがあります。

```txt
プロフィール画像
PDF
動画
アップロードファイル
```

DBに画像ファイルそのものを入れるのではなく、ファイル本体はストレージに置き、DBにはURLやファイル名などのメタ情報を保存することがあります。

| 保存するもの | 置き場所の例 |
| --- | --- |
| ユーザー名、メールアドレス | DB |
| プロフィール画像のURL | DB |
| プロフィール画像のファイル本体 | ストレージ |

## DBとストレージの使い分け

DBとストレージは、保存するものの性質で使い分けます。

| 使いたいもの | 向いている保存先 |
| --- | --- |
| 検索・集計・更新したいデータ | DB |
| 行や列で整理したいデータ | DB |
| 画像やPDFなどのファイル本体 | ストレージ |
| 大きな動画や添付ファイル | ストレージ |

実際のサービスでは、DBとストレージを組み合わせて使うことが多いです。

```txt
DB: ファイル名、URL、誰がアップロードしたか
ストレージ: ファイル本体
```

## まず押さえること

最初に押さえるべきことは次の4つです。

- DBは、検索・更新しやすいデータを保存する場所
- SQLは、DBからデータを取得・追加・更新・削除するための言語
- CRUDは、Create / Read / Update / Delete の基本操作
- ストレージは、画像やPDFなどのファイル本体を保存する場所

> DBとストレージの違いが分かると、データがどこに保存されているかを考えやすくなります。

## 理解度チェック

Q1. DBの説明として最も近いものはどれですか。

- A. 画像ファイルだけを圧縮するツール
- B. 構造化されたデータを保存・検索・更新するための仕組み
- C. Gitのブランチを切り替えるためのコマンド
- D. DNSの名前解決だけを行うサービス

解説: DBは、ユーザー情報や注文情報など、検索・更新しやすい形でデータを保存する仕組みです。

Q2. SQLの役割として最も近いものはどれですか。

- A. データベースに問い合わせて、データを取得・追加・更新・削除するための言語
- B. 画像をブラウザに表示するためだけの形式
- C. HTTPSの暗号化を無効化するコマンド
- D. npmパッケージをインストールするためのファイル

解説: SQLは、`SELECT` や `UPDATE` などを使って、データベースを操作するための言語です。

Q3. CRUDのうち、既存データを読み取る操作にあたるものはどれですか。

- A. Create
- B. Update
- C. Read
- D. Delete

解説: CRUDのReadは、保存されているデータを読み取る操作です。SQLでは `SELECT` が対応します。

Q4. ログイン情報の扱いとして適切な説明はどれですか。

- A. パスワードは必ずそのままの文字列でDBに保存する
- B. メールアドレスやパスワードはDBと一切関係しない
- C. ログイン処理ではSQLを使うことは絶対にない
- D. ログインでは、入力された情報をもとにDBからユーザーを探すことがある

解説: ログインでは、メールアドレスなどをもとにユーザーを検索し、入力されたパスワードと保存済みのハッシュを照合することがあります。

Q5. プロフィール画像のようなファイル本体の保存先としてよく使われるものはどれですか。

- A. Gitのコミットメッセージ
- B. HTTPステータスコード
- C. DNSレコード
- D. ストレージ

解説: 画像やPDFなどのファイル本体はストレージに置き、DBにはURLやメタ情報を保存することがあります。

答え:

- Q1: B
- Q2: A
- Q3: C
- Q4: D
- Q5: D
