# LLMとは何か

LLMは、Large Language Modelの略です。日本語では大規模言語モデルと呼ばれます。

大量の文章やコードを学習し、入力された文章に続く自然な出力を生成するモデルです。

```txt
入力された文章
  ↓
LLM
  ↓
次に来そうな文章を生成
```

> まとめ: LLMは、大量の言語データから学習し、文章やコードを生成するモデルです。

## 言語モデルとは

言語モデルは、言葉の並び方のパターンを学ぶモデルです。

言い換えると、LLMは文脈を見て「次に来そうな言葉」を予測しています。

たとえば、次の文があるとします。

```txt
今日は天気が
```

このあとには、「いい」「悪い」「よい」などの言葉が続きやすいと考えられます。

LLMは、もっと大きな規模で、文脈に合う言葉の続き方を学習しています。

ただし、LLMは文章をそのまま人間のように読んでいるわけではありません。

文章を小さな単位に分け、数値として扱います。この小さな単位を **トークン** と呼びます。

```txt
入力: 生成AIを使う
  ↓
トークン: 生成 / AI / を / 使う
```

実際の分かれ方はモデルによって異なります。日本語では、1文字、単語、単語の一部のような単位に分かれることがあります。

LLMは、このトークンの並びを見て「次に来そうなトークン」を予測します。

```txt
今日は天気が
  ↓
次に来そうなトークンを予測
  ↓
いい / よい / 悪い ...
```

この予測を繰り返すことで、文章が少しずつ生成されます。

```txt
1つ目のトークンを予測
  ↓
次のトークンを予測
  ↓
さらに次のトークンを予測
  ↓
文章になる
```

では、どうやって次に来そうなトークンを選ぶのでしょうか。

LLMは、トークンを数値のベクトルとして扱います。言葉や文章の意味を数値のベクトルとして表したものを **埋め込み** と呼びます。

埋め込みにすると、言葉同士の近さや関連の強さを計算しやすくなります。

たとえば、「犬」と「猫」はどちらも動物なので意味が近く、「自動車」とは少し離れていると考えられます。

このような関連の強さを計算するときに、内積のような計算が使われます。

```txt
トークンAのベクトル
  ×
トークンBのベクトル
  ↓
関連が強いかを計算
```

初学者のうちは、内積の数式を覚える必要はありません。

「LLMは文章をトークンに分け、トークンを数値ベクトルに変換し、関連が強い候補を見ながら次に来そうなトークンを予測している」と考えると理解しやすいです。

## ニューラルネットワークとの関係

LLMは、ニューラルネットワークを大規模にした技術の一種です。

ニューラルネットワークは、人間の脳の神経細胞のつながり方をヒントにした仕組みです。

ただし、人間の脳をそのまま再現したものではありません。入力されたデータを数値として受け取り、内部のパラメータを通して出力を計算する仕組みです。

```txt
入力データ
  ↓
たくさんのパラメータで計算
  ↓
出力
```

機械学習のレッスンで見たように、学習では予測と正解の差が小さくなるようにパラメータを調整します。

LLMの場合は、文章やコードを大量に学習し、「この文脈なら次にどの言葉が来やすいか」を計算できるようにパラメータを調整しています。

## RNNとの違い

RNNは、Recurrent Neural Networkの略です。日本語では再帰型ニューラルネットワークと呼ばれます。

RNNは、文章や時系列データのように、順番があるデータを扱うために使われてきたニューラルネットワークです。

ただし、RNNは前から順番に情報を渡していくため、文が長くなると前の方の情報が薄れやすいという課題があります。

現在の多くのLLMは、Transformerという構造を土台にしています。Transformerは、文章全体の中でどの単語同士が関係しているかを見やすく、長い文脈を扱いやすい点が特徴です。

| モデル | 得意なこと | 弱点・特徴 |
| --- | --- | --- |
| RNN | 順番のあるデータを前から処理する | 長い文では前の情報が薄れやすい |
| Transformer系のLLM | 文中の単語同士の関係を見ながら処理する | 文脈を扱いやすいが、必ず正しいとは限らない |

RNNもLLMも、広い意味ではニューラルネットワークを使った技術です。

違いは「ニューラルネットワークかどうか」ではなく、文章の中の情報をどのように扱うかです。

> まずは、RNNは順番に情報を渡すモデル、Transformer系のLLMは文中の関係を広く見ながら処理するモデル、と押さえると理解しやすいです。

## コンテキストとは

LLMは、入力された会話や文章の範囲をもとに回答します。

このモデルが回答時に参照できる文脈の範囲を、コンテキストと呼ぶことがあります。

| 言葉 | 意味 |
| --- | --- |
| プロンプト | LLMに渡す指示や入力 |
| コンテキスト | LLMが回答時に参照できる文脈 |
| 出力 | LLMが生成した回答 |

よい回答を得るには、必要な前提や条件をコンテキストに入れることが重要です。

## コンテキスト長

LLMには、一度に扱えるトークン数に上限があります。

これを文脈長、またはコンテキスト長と呼ぶことがあります。

| 観点 | 意味 |
| --- | --- |
| 入力トークン | プロンプトや添付情報として渡す量 |
| 出力トークン | AIが生成する回答の量 |
| コンテキスト長 | 入力と出力を含めて扱える文脈の上限 |

長い資料をすべて入れようとすると、上限を超えることがあります。その場合は、要約したり、必要な部分だけ渡したりします。

## LLMができること

LLMは、文章を扱うさまざまな作業に使われます。

| 作業 | 例 |
| --- | --- |
| 文章生成 | メール、説明文、記事の下書き |
| 要約 | 長い文章を短くまとめる |
| 翻訳 | 日本語から英語へ変換する |
| 質問応答 | 入力された質問に答える |
| コード生成 | 関数やテストコードを書く |
| 分類 | 問い合わせ内容をカテゴリ分けする |

LLMは、言葉を使った作業の補助に向いています。

## LLMは理解しているのか

LLMは、人間と同じように世界を理解しているわけではありません。

大量のデータから言葉の関係やパターンを学び、文脈に合う出力を生成しています。

LLMは、基本的には文脈から次に来そうなトークンを予測し続ける仕組みです。

そのため、文章として自然に見えても、内容が事実と違うことがあります。

このように、もっともらしい誤情報を出すことを **ハルシネーション** と呼ぶことがあります。

```txt
自然な文章
  ↓
必ず正しいとは限らない
```

LLMの出力は、特に事実確認や専門判断が必要な場面では確認が必要です。

## まず押さえること

- LLMは大規模言語モデル
- LLMはニューラルネットワークを大規模にした技術の一種
- 大量の文章やコードから言葉のパターンを学ぶ
- RNNは長い文脈で前の情報が薄れやすく、Transformer系のLLMは文中の関係を扱いやすい
- LLMは文章をトークンに分け、次に来そうなトークンを予測する
- 埋め込みは、トークンや文章を数値ベクトルとして扱うための表現
- 要約、文章作成、質問応答、コード生成などに使える
- 出力は自然でも、必ず正しいとは限らない

> LLMは言葉を扱う強力なモデルですが、出力を確認しながら使うことが重要です。

## 理解度チェック

Q1. LLMの説明として最も近いものはどれですか。

- A. 大量の文章やコードから学習した大規模な言語モデル
- B. 画像ファイルだけを圧縮する仕組み
- C. DNSの名前解決を行うレコード
- D. EC2の台数を増やすロードバランサー

解説: LLMはLarge Language Modelの略で、大量の言語データから学習したモデルです。

Q2. LLMが文章を生成する基本的な考え方として最も近いものはどれですか。

- A. DNSレコードを順番に並べ替える
- B. 物理サーバーの配線を自動で変更する
- C. 入力文をトークンに分け、次に来そうなトークンを予測していく
- D. Gitのコミット履歴だけを参照して回答する

解説: LLMは文章をトークンに分け、文脈から次に来そうなトークンを予測することで文章を生成します。

Q3. RNNとTransformer系LLMの違いとして最も近いものはどれですか。

- A. RNNはニューラルネットワークではなく、LLMだけがニューラルネットワークである
- B. RNNは順番に情報を渡すため長い文脈で前の情報が薄れやすく、Transformer系LLMは文中の関係を扱いやすい
- C. RNNは画像だけを扱い、LLMは音声だけを扱う
- D. RNNとTransformer系LLMは仕組みも得意なことも完全に同じである

解説: RNNもニューラルネットワークの一種ですが、順番に情報を渡すため長い文脈では前の情報が薄れやすい課題があります。Transformer系LLMは文中の関係を扱いやすい構造です。

Q4. LLMでハルシネーションが起きる理由として最も近いものはどれですか。

- A. LLMは常に公式情報だけを参照しているため
- B. LLMは文脈から自然に続きそうなトークンを予測するため、自然な文章でも事実と違うことがあるため
- C. LLMは文章を一切生成できないため
- D. LLMはコンテキストをまったく使わないため

解説: LLMは次に来そうなトークンを予測して文章を作るため、自然な文章でも事実と違う内容になることがあります。

Q5. コンテキストの説明として最も近いものはどれですか。

- A. S3に保存する画像ファイル
- B. HTTPの標準ポート番号
- C. Gitのブランチ名
- D. LLMが回答時に参照できる文脈

解説: コンテキストは、LLMが回答するときに参照できる会話や文章の範囲です。

Q6. トークンと埋め込みの説明として最も近いものはどれですか。

- A. トークンはDNSの設定で、埋め込みはGitのブランチ名である
- B. トークンも埋め込みもLLMとは関係しない
- C. トークンは画像ファイルの保存場所で、埋め込みはサーバーの電源である
- D. トークンは文章を処理するための小さな単位で、埋め込みはそれを数値ベクトルとして扱う表現

解説: LLMは文章をトークンに分け、埋め込みによって数値ベクトルとして扱いながら、次に来そうなトークンを予測します。

- Q1: A
- Q2: C
- Q3: B
- Q4: B
