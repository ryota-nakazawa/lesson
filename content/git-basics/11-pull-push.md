# リモートとやりとりする: pull / push

Gitでは、自分のPCだけでなく、リモートリポジトリと変更をやりとりします。

代表的なコマンドが `git pull` と `git push` です。

> まとめ: `pull` はリモートから取り込む操作、`push` はリモートへ送る操作です。

## ローカルとリモートの対応

Gitでは、自分のPCにあるリポジトリを **ローカルリポジトリ**、GitHubなどにある共有用のリポジトリを **リモートリポジトリ** と呼びます。

ローカルとリモートでは、基本的に同じ名前のブランチを対応させて扱います。

```txt
ローカル: main
リモート: origin/main
```

たとえば、自分のPCで `main` ブランチを作業しているとき、リモート側にも `main` ブランチがあります。Gitでは、リモート側の `main` を `origin/main` のように表すことがあります。

ここで出てくる `origin` は、リモートリポジトリに付けられた標準的な名前です。多くの場合、最初にGitHubからリポジトリを取得したとき、リモートリポジトリには自動的に `origin` という名前が付きます。

> ポイント: `origin/main` は、「originというリモートにあるmainブランチ」と考えると分かりやすいです。

## git pull

`git pull` は、リモートリポジトリの最新状態を自分のローカルリポジトリへ取り込むコマンドです。

```bash
git pull
```

対象を明示して書くこともできます。

```bash
git pull origin main
```

これは、「`origin` というリモートにある `main` ブランチの変更を取り込む」という意味です。

チームで作業していると、自分が作業している間に他の人の変更がリモートへ入っていることがあります。

作業前や作業の区切りで `git pull` すると、手元を新しい状態に近づけられます。

## git push

`git push` は、自分のローカルリポジトリで作ったコミットをリモートリポジトリへ送るコマンドです。

```bash
git push
```

対象を明示して書くこともできます。

```bash
git push origin main
```

これは、「自分のローカルの `main` ブランチのコミットを、`origin` というリモートの `main` ブランチへ送る」という意味です。

ローカルでコミットしただけでは、まだチームには共有されていません。`git push` してはじめて、リモート側でもその変更を見られるようになります。

## pull と push の向き

向きで考えると分かりやすくなります。

```txt
リモート → 自分のPC: git pull
自分のPC → リモート: git push
```

`pull` は受け取る、`push` は送る、と覚えると整理しやすいです。

`origin main` を付けて考えると、向きはさらに具体的になります。

```txt
origin/main → ローカルのmain: git pull origin main
ローカルのmain → origin/main: git push origin main
```

普段は `git pull` や `git push` だけで動くことも多いですが、初学者のうちは「どのリモートの、どのブランチとやりとりしているか」を意識すると理解しやすくなります。

## push したあとの流れ

チーム開発では、作業ブランチを `push` したあとにPull Requestを作ることがよくあります。

- ローカルで作業する
- コミットする
- リモートへpushする
- Pull Requestを作る
- レビューしてもらう

> pushはゴールではありません。チームで確認できる場所に変更を置く操作です。

## 理解度チェック

Q1. `origin` の説明として最も近いものはどれですか。

- A. ローカルで作ったコミットメッセージ
- B. リモートリポジトリに付けられた標準的な名前
- C. ステージングに置かれた変更
- D. GitHubのレビューコメント

解説: `origin` は、GitHubなどにあるリモートリポジトリに付けられる標準的な名前です。

Q2. `origin/main` の説明として最も近いものはどれですか。

- A. ローカルにある作業フォルダ
- B. コミット前の一時置き場
- C. GitHubのアカウント名
- D. `origin` というリモートにある `main` ブランチ

解説: `origin/main` は、「originというリモートにあるmainブランチ」と考えると分かりやすいです。

Q3. `git pull origin main` は何をするコマンドですか。

- A. `origin` の `main` ブランチの変更をローカルへ取り込む
- B. ローカルの `main` ブランチをリモートへ送る
- C. 新しいブランチを作る
- D. コミットメッセージを変更する

解説: `git pull origin main` は、リモートの `origin/main` から変更を取り込む操作です。

Q4. `git push origin main` は何をするコマンドですか。

- A. リモートの変更をローカルへ取り込む
- B. ステージングの変更を削除する
- C. ローカルの `main` ブランチのコミットを `origin` の `main` ブランチへ送る
- D. Pull Requestを自動でレビューする

解説: `git push origin main` は、ローカルの `main` ブランチのコミットをリモートの `origin/main` へ送る操作です。

答え:

- Q1: B
- Q2: D
- Q3: A
- Q4: C
